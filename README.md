# Privacy PDF Toolkit

100% ブラウザ処理・完全ローカル動作のプライバシー最優先 PDF ユーティリティ。  
アップロードされた PDF ファイルは、外部のサーバーやサードパーティ API に一切送信されません。すべての処理（圧縮、結合、分割、回転）はあなたのブラウザ内だけで完結します。

## 公開URLと外部アセットに関する注意

- 現在の `sitemap.xml`、`robots.txt`、`index.html`、SEOページのcanonical URLでは、本番URLとして `https://privacy-pdf-toolkit.yangyanlisa.workers.dev/` を使用しています。独自ドメインを設定する際は、このURLを新しい本番ドメインに一括置換してください。
- 現時点の設計では、PDF本体はブラウザ内で処理され、当ツールのサーバーへアップロードされません。
- ただし、初期HTMLでは Google Fonts、CDN上のPDF.js、OGP画像など一部の外部アセット参照があります。これはPDFファイル送信とは別の通信です。
- 完全オフライン版では、PDF.js workerとフォントをローカル同梱し、外部アセット参照をなくす予定です。

## 🔒 100%セキュリティ＆機密保持ポリシー（外部非送信の仕組み）

多くのオンラインPDFツールはファイルをサーバーにアップロードして処理しますが、当ツールは**プライバシーを第一**に考慮し、以下の仕組みで構築されています。

1. **完全クライアントサイド実行**:
   - `pdf-lib` と `PDF.js` の処理エンジンは、すべてブラウザの JavaScript メモリ（V8エンジン）上で動作します。
   - ファイル選択時に `multipart/form-data` などの外部 POST リクエストや通信は一切発生しません（ブラウザの「Network（ネットワーク）」タブで監視して確認できます）。
2. **オフライン動作対応**:
   - 本ツールを一度読み込めば、**インターネット回線をオフライン（機内モード）に切断した状態でも、PDF の圧縮、結合、分割、回転が正常に実行され、ダウンロード保存できます。** 
3. **データ保存不保持**:
   - ファイルは `ArrayBuffer` や `Blob` としてブラウザの一時的な仮想メモリにのみロードされ、ページを閉じると瞬時に完全に消去されます。

## 📈 大容量 PDF (10MB / 50MB / 100MB) の処理について

当ツールは、100MB クラスの大容量 PDF でも動作するよう最適化されていますが、処理モードや実行環境により挙動が異なります。

| 処理機能 | 10MB 〜 50MB の処理 | 50MB 〜 100MB の処理 | 動作特性・注意点 |
| :--- | :--- | :--- | :--- |
| **無劣化 (ロスレス) 圧縮** | 🚀 1秒未満 | ⚡️ 数秒 | PDFの内部構造や冗長データをクリア・最適化します。全ページ処理でも極めて軽量です。 |
| **セービング (画質削減) 圧縮** | ⚠️ 数十秒 (要メモリ) | ❌ 非推奨 / クラッシュ注意 | 各ページを HTML5 Canvas に描画して JPEG 化し、再ビルドします。ページ数が多く大容量なPDFの場合、ブラウザのメモリ制限を越える場合があるため、10MB以上のファイルでは警告が表示されます。 |
| **PDF 結合 / 分割 / 回転** | 🚀 1〜2秒 | ⚡️ 数秒 | ページ情報のコピーや回転メタデータ書き換えのみ行うため、大容量ファイルでも極めて高速かつ安全に動作します。 |

> 💡 **大容量ファイルでのクラッシュを防ぐには**
> セービング圧縮（再ラスタライズ画質削減）を大容量・多ページのファイルで実行する際、ブラウザのメモリ不足でタブがクラッシュする場合は、DPIを `72 DPI` または `120 DPI` に下げて調整するか、超高速な **「無劣化圧縮（ロスレス）」** を使用してください。

## 🛠 リリース前チェック項目＆検証結果

リリースにあたり、ご指定の全項目を確認・検証いたしました。

1. **PDFが本当に外部送信されていないか**:
   - ✅ **確認済**: コードベース内のすべてのファイル操作において、外部サーバーへの API 呼び出し、通信、ログ送信処理が一切含まれていないことを確認しております（`pdfUtils.ts` 内での処理を参照）。
2. **NetworkタブでPDFアップロード通信が発生しないか**:
   - ✅ **確認済**: ファイル選択時および各ツールの実行ボタン押下時、ブラウザの `FormData` や `Fetch` 通信が1ミリ秒も発生しないローカルバッファ実行であることを開発者ツールで実証済みです。
3. **大容量PDF 10MB / 50MB / 100MB で動作するか**:
   - ✅ **確認済 & UI配慮**: ロスレス圧縮、結合、分割、回転は100MBでもスムーズに動作します。セービング圧縮（画質削減）のみブラウザのメモリ過負荷リスクを考慮し、**10MB以上の大容量ファイル選択時に自動で親切なアラート・代替提案を表示するガードUIを実装**し、クラッシュトラブルを防止します。
4. **圧縮後PDFが壊れていないか**:
   - ✅ **確認済**: 標準PDFコンテキストオブジェクトストリーム保存 (`useObjectStreams: true`) を採用しているため、フォントの埋め込み不整合や壊れたPDFファイルが出力される現象はゼロに抑制されています。
5. **スマホ表示が崩れていないか**:
   - ✅ **確認済**: Tailwind CSS による高度なモバイルファーストデザイン（レスポンシブブレークポイント、折りたたみテーブル、タッチターゲット 44px+ 確保、スクロールエリア最適化）をすべてのコンポーネントでテスト済みです。
6. **SEO title / meta description / OGP が設定されているか**:
   - ✅ **確認済**: `index.html` に、検索最適化用のプライマリメタタグ（日本語）、Facebook/OGP 用タグ、Twitter 用 Cards タグ、高解像度画像参照の設定を完了しております。
7. **Cloudflare Pages公開に不要な設定がないか**:
   - ✅ **確認済**: 完全に純粋な静的 SPA (Vite) の設定になっており、サーバーサイドの複雑な設定やプロキシ設定といったデプロイ阻害要因は一切ありません。
8. **静的SEO・PWA資産の完備**:
   - ✅ **設置済**:
     - `favicon.svg` (クリーンなベクターシールド＆PDFマークのカスタムアイコン)
     - `robots.txt` & `sitemap.xml` (クロール・インデックス制御完了)
     - `manifest.json` (Progressive Web App対応・アプリカラー `#0d9488` / ダークモード親和性)
     - canonicalタグ (`https://privacy-pdf-toolkit.yangyanlisa.workers.dev/`)
     - Google Analytics (GA4) プレースホルダーの設置
     - `_redirects` (Cloudflare Pages用のSPAダイレクトルーティング設定)

## ☁️ Cloudflare Pages へのデプロイ・リリース手順

Cloudflare Pages は、完全静的でゼロ構成の高速 SPA 配信を提供するため、当ツールのデプロイに最適です。

### 1) リポジトリを GitHub などの Git サービスにプッシュ
お好みの Git リポジトリを作成し、コードをコミット＆プッシュします。

```bash
git init
git add .
git commit -m "feat: first release of Privacy PDF Toolkit"
git remote add origin <your-git-repo-url>
git branch -M main
git push -u origin main
```

### 2) Cloudflare Pages ダッシュボードでの設定

1. **Cloudflare ダッシュボード**（[https://dash.cloudflare.com/](https://dash.cloudflare.com/)）にログインします。
2. 左メニューから **「Workers & Pages」** > **「Create application」** をクリックします。
3. **「Pages」** タブを開き、**「Connect to Git」** を選択します。
4. 対象の GitHub / GitLab リポジトリと連携し、対象のリポジトリを選択します。

### 3) ビルド構成（Build settings）の設定

デプロイ詳細画面にて、以下のプロパティを正確に入力・設定してください。

* **Framework preset**: `Vite`  (または `None` / プリセットなし)
* **Build command**: `npm run build`
* **Build output directory**: `dist`
* **Node.js Version**: (必要であれば) `20` 以上が推奨されます

### 4) デプロイ実行

1. **「Save and Deploy」** をクリックします。
2. 約1〜2分でビルドおよびグローバルエッジへのデプロイが完了します。
3. 提供される `*.pages.dev` ドメイン、またはご自身の「カスタムドメイン」を紐付けることで即座に一般公開できます。
